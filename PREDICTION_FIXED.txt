================================================================================
                          TROUBLESHOOTING
================================================================================

If prediction still fails:

1. Check browser console (F12) for errors
2. Check Network tab to see API calls
3. Verify URL shows: http://localhost:8000/api/v1/predictions/upload
4. Check backend terminal for errors
5. Try uploading a small image (< 1MB)

If "404 Not Found":
- Make sure backend is running on port 8000
- Check: http://localhost:8000/docs

If "500 Internal Server Error":
- Check backend terminal for Python errors
- Verify uploads/ folder exists in backend/

If upload works but analyze fails:
- Check disease IDs in database
- Verify seed.py was run
- Check recommendation engine

================================================================================
                          WORKFLOW
================================================================================

Complete Upload → Analyze → View Workflow:

1. **Login** (http://localhost:3001/auth/login)
   - Use farmer@example.com / farmer123

2. **Upload** (http://localhost:3001/scan)
   - Drag & drop image OR click to browse
   - Multiple images supported (up to 50)
   - Max 15MB per file

3. **Analyze**
   - Click "Analyze" button
   - Backend uploads to /uploads/ folder
   - ML service returns random disease
   - Recommendation engine finds treatments
   - Creates prediction record in database

4. **View Results** (http://localhost:3001/history)
   - See all predictions
   - Click on prediction for details
   - View disease info
   - See pesticide recommendations
   - See fertilizer recommendations

5. **Download PDF** (http://localhost:3001/history/{id})
   - Click "Download Report" button
   - PDF generated with ReportLab
   - Includes image, disease, recommendations
